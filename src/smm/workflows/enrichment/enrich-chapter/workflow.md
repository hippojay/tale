# Enrich Chapter Workflow

## Objective
Add descriptive depth, sensory detail, and immersive world-building to existing chapter content without disrupting pacing or voice.

## Prerequisites
- Completed chapter draft
- Story Bible (story-bible.md) for world consistency
- Story Requirements Document (SRD) for tone and style

## Workflow Steps

### 1. Initial Assessment
- Read chapter completely to understand flow and pacing
- Load story-bible.md for established sensory palette and world details
- Identify scenes that feel thin or generic
- Note areas where reader might struggle to visualize
- Assess current sensory balance (sight, sound, smell, touch, taste)

### 2. Setting Enrichment
- Deepen location descriptions with specific, evocative details
- Add atmospheric elements (weather, lighting, ambient sounds)
- Include historical or cultural context where appropriate
- Ensure settings feel lived-in and real
- Maintain consistency with established world details

### 3. Character Presence
- Enhance physical descriptions that reveal personality
- Add character mannerisms and body language
- Include emotional physical responses (tension, relaxation, etc.)
- Show character interaction with environment
- Deepen POV character's sensory experience

### 4. Sensory Balance
- Audit each scene for sensory engagement
- Add non-visual senses where missing (sound, smell, touch, taste)
- Use specific sensory details over generic ones
- Connect sensory details to emotion and memory
- Avoid sensory overload - balance with action and dialogue

### 5. Metaphor and Imagery
- Add purposeful metaphors that illuminate character or theme
- Use similes that feel natural to POV character's voice
- Ensure figurative language enhances rather than distracts
- Keep imagery consistent with genre and tone

### 6. Pacing Protection
- Ensure enrichment doesn't slow action scenes
- Balance description with dialogue and movement
- Keep paragraphs varied in length
- Maintain chapter momentum and flow

### 7. Quality Check
- Read enriched chapter for natural flow
- Verify no purple prose or overwrought description
- Confirm consistency with story bible
- Check that voice remains consistent
- Ensure enrichment serves story, not just decoration

### 8. Documentation
- Note any new world details added
- Update story bible if significant details introduced
- Flag any continuity questions
- Provide summary of enrichment focus areas

## Output Format
- Enriched chapter: `{output_folder}/chapters/chapter-{number}-enriched.md`
- Track changes document if requested
- Summary of enrichment additions

## Notes for Agent
- Less is often more - be selective and purposeful
- Specific details beat generic descriptions every time
- Respect the original pacing and voice
- When in doubt, check story bible for consistency
- Description should emerge naturally through character POV
