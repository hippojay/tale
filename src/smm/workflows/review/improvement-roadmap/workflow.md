# Improvement Roadmap Workflow

## Objective
Create prioritized list of improvements with specific actions.

## Prerequisites
- Review feedback compiled
- Understanding of story goals

## Steps

### 1. Compile All Issues
- Gather from all reviews
- Include self-identified issues
- Add beta reader feedback

### 2. Categorize
- Structure
- Character
- Pacing
- Dialogue
- Description
- Continuity
- Craft

### 3. Prioritize
**Critical:** Must fix
**Important:** Should fix
**Nice-to-have:** Could fix

### 4. Create Action Items
For each issue:
- Specific action to take
- Location in manuscript
- Estimated effort
- Dependencies

### 5. Sequence Actions
- Structure first
- Character second
- Scene-level third
- Line-level last

### 6. Group into Passes
- Pass 1: Critical structural
- Pass 2: Character and arc
- Pass 3: Scene improvements
- Pass 4: Polish

### 7. Set Checkpoints
- Review after each pass
- Verify fixes don't create new issues
- Update roadmap as needed

### 8. Create Checklist
Actionable format:
- [ ] Action item (Chapter X)
- [ ] Action item (Chapter Y)

## Output
- Prioritized improvement roadmap
- Pass-by-pass checklist
- Timeline estimate
